# 把区间操作变成差分再做 DP 的常用技巧

把每次对区间 $[l,r]$ 加 $+1$ 看成在差分数组 $b$ 上把 $b[l]\,+\,1,\,b[r\,+\,1] \,-\,1$，最终每个位置被增加的总次数记为 $inc[i]$（即目标值减原值），因此总操作天数恰好等于差分中正项之和 $$\sum_i\max(0,\,b[i])=\sum_i\max(0,\,inc[i]-inc[i-1])$$（约定 $inc[0]=0$）

DP：令 $f[i][s]$ 表示处理到 $i$ 且 $i$ 选状态 $s$ 时的最小累计开始数，转移为 $$f[i+1][t]=\min_s\{f[i][s]+\max(0,\,inc_t[i+1]-inc_s[i])\}$$因为只有当后者大于前者时才需“新开始”若干区间（所以用加法累积，而非取 max）；

枚举所有候选高度并取最小即可。
